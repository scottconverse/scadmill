# Security policy

## Supported versions

Security fixes target the latest public Windows beta, currently `0.1.0-beta.2`. Older beta builds and unpublished GitHub Actions artifacts are not supported distributions.

## Report a vulnerability privately

Use GitHub's [private vulnerability report](https://github.com/scottconverse/scadmill/security/advisories/new) for suspected security vulnerabilities in ScadMill. Do not open a public issue or discussion for a vulnerability that could put users, projects, credentials, or systems at risk.

Include the affected ScadMill version, operating system, impact, reproduction steps, and any suggested mitigation. Attach the smallest safe proof of concept needed to reproduce the problem. Do not include API keys, access tokens, passwords, private project files, or other secrets; redact sensitive values from logs and screenshots.

Reports are assessed privately and may be declined when they are not security issues or do not affect ScadMill. When a confirmed issue requires public disclosure, the maintainers may publish a GitHub security advisory after a fix or mitigation is available.

ScadMill does not currently offer a bug bounty and does not promise a response or remediation deadline.
